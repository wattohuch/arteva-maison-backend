# 🚀 Deploy New Receipt System

## Quick Deployment Guide

### Step 1: Commit Changes

```bash
cd arteva-maison-backend

# Add new files
git add src/utils/receiptTemplate.js
git add src/controllers/adminController.js
git add RECEIPT_REDESIGN_COMPLETE.md
git add RECEIPT_IMPLEMENTATION_CHECKLIST.md
git add DEPLOY_NEW_RECEIPT.md

# Commit
git commit -m "feat: pixel-perfect A4 receipt redesign for print station

- Created new receiptTemplate.js with exact reference design match
- Updated adminController to use new template
- Removed old receipt generation function
- Added comprehensive documentation
- 100% compatible with existing print station
- No breaking changes"

# Push to main
git push origin main
```

### Step 2: Deploy to Render

The backend will automatically deploy when you push to main. Monitor the deployment:

1. Go to https://dashboard.render.com
2. Find "arteva-maison-backend" service
3. Watch the deployment logs
4. Wait for "Deploy succeeded" message

### Step 3: Verify Deployment

Test the new receipt endpoint:

```bash
# Replace ORDER_ID with a real paid order ID
# Replace TOKEN with your superuser token
curl -H "Authorization: Bearer TOKEN" \
  https://arteva-maison-backend.onrender.com/api/admin/receipt/ORDER_ID \
  > test-receipt.html

# Open test-receipt.html in browser to verify
```

### Step 4: Print Station (No Changes Needed!)

The Raspberry Pi print station will automatically use the new receipt design. No restart or updates required!

**Why?** The print station fetches HTML from the backend API, so it automatically gets the new design once the backend is deployed.

### Step 5: Test Print

#### Option A: Test Mode (Recommended First)
```bash
# SSH into Raspberry Pi
ssh pi@your-raspberry-pi-ip

# Navigate to print station
cd /home/pi/arteva-print-station

# Run test print
TEST_MODE=true node print-station-hp.js
```

This will print a test receipt with sample data.

#### Option B: Real Order Test
1. Place a test order on the website
2. Complete payment
3. Print station will automatically print the new receipt
4. Verify the output matches the reference design

## 🐛 Troubleshooting

### Issue: Backend deployment fails

**Solution:**
```bash
# Check for syntax errors
cd arteva-maison-backend
npm install
node -c src/utils/receiptTemplate.js
node -c src/controllers/adminController.js
```

### Issue: Receipt endpoint returns old design

**Solution:**
1. Clear Render build cache
2. Redeploy from Render dashboard
3. Wait 2-3 minutes for deployment to complete
4. Test again

### Issue: Print station not printing new design

**Solution:**
```bash
# SSH into Raspberry Pi
ssh pi@your-raspberry-pi-ip

# Check print station logs
tail -f /home/pi/arteva-print-station/logs/print-station.log

# Restart print station service
sudo systemctl restart arteva-print-station

# Check status
sudo systemctl status arteva-print-station
```

### Issue: QR code not working

**Solution:**
- QR code is generated by external API (qrserver.com)
- Check internet connection on Raspberry Pi
- Verify QR URL is correct in generated HTML
- Test QR URL manually in browser

### Issue: Arabic text not displaying

**Solution:**
- Ensure Google Fonts are loading (check network)
- Verify Noto Sans Arabic font is included
- Check RTL direction is applied
- Test in browser first before printing

### Issue: Receipt doesn't fit on one page

**Solution:**
- Check @page margin settings (should be 10mm)
- Verify paper size is A4 in printer settings
- Check print station PDF generation settings
- Ensure no extra margins added by printer driver
